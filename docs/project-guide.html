1. Project Overview

FreshCart is a full-stack grocery e-commerce platform built with Next.js 14. It has a customer-facing storefront, a full admin dashboard, user authentication, order management, real-time stock updates, and PWA capabilities with push notifications.

🛍️ Shop Features
  • Product browsing with filters
  • Infinite scroll
  • Cart with sidebar
  • Checkout + order placement
  • Order history with confetti
  • Pincode delivery check
  • Command palette (⌘K)
⚙️ Admin Features
  • Dashboard with live stats
  • Analytics with charts
  • Product CRUD + image fix
  • Order management + status
  • Category management
  • Pincode management
  • Push notification sender
🔐 Auth & UX
  • Email/password auth (NextAuth)
  • Role-based access control
  • Dark / Light / System theme
  • Shimmer skeleton loading
  • Animated empty states
  • PWA + Push Notifications
  • Mobile BottomNav

4. How It Works — Key Flows

🔐 Authentication Flow

User submits login form → NextAuth CredentialsProvider → bcrypt.compare() → JWT cookie set → Redirect by role

Admin users are redirected to /admin/dashboard. Regular users go to the homepage or their requested page.

🛒 Cart → Order Flow

Add to Cart → Zustand Store → CartSidebar opens → Checkout form → POST /api/orders → MongoDB Order saved → Confetti 🎉

🔍 Search Flow

User types → 300ms debounce → GET /api/products?search=… → MongoDB $regex query → Dropdown results

📦 Admin Order Management

pending → confirmed → out_for_delivery → delivered

Admin clicks "→ Confirmed" button in the order detail modal → PATCH /api/admin/orders → MongoDB status updated.

🌙 Dark Mode Flow

ThemeToggle click → ThemeProvider updates state → Saves to localStorage → .dark class on <html> → CSS vars switch → All dark: classes activate

5. Setup & Running Locally

Prerequisites

  • Node.js 18+ and npm
  • MongoDB database (local or MongoDB Atlas free tier)
  • Git

Step 1 — Clone and Install

git clone https://github.com/sumit-verma11/freshCart.git cd freshCart npm install

Step 2 — Environment Variables

Create a .env.local file in the project root:

# MongoDB MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/freshcart # NextAuth NEXTAUTH_SECRET=any-long-random-string-here NEXTAUTH_URL=http://localhost:3000 # Web Push VAPID Keys (generate with: npx web-push generate-vapid-keys) VAPID_PUBLIC_KEY=your-public-key VAPID_PRIVATE_KEY=your-private-key # Optional: Unsplash for product images UNSPLASH_ACCESS_KEY=your-unsplash-api-key

Step 3 — Seed the Database

npm run seed # seeds products, categories, pincodes, users

This creates demo accounts:

RoleEmailPassword
Adminadmin@freshcart.inAdmin@1234
Userarjun@example.comUser@1234

Step 4 — Run Development Server

npm run dev

App runs at http://localhost:3000

Build for Production

npm run build npm run start

9. Deployment (Vercel)

Deploy to Vercel

  1. Push code to GitHub (git push origin main)
  2. Go to vercel.com → Import Project from GitHub
  3. Select the freshCart repo
  4. Add all environment variables from .env.local in the Vercel dashboard
  5. Click Deploy — Vercel auto-detects Next.js and builds it
⚠️ Make sure to update NEXTAUTH_URL to your Vercel domain (e.g., https://freshcart.vercel.app) and whitelist the domain in your MongoDB Atlas Network Access settings.

MongoDB Atlas (Free Tier)

  1. Create account at mongodb.com/atlas
  2. Create a free M0 cluster
  3. Create a database user with read/write permissions
  4. Add 0.0.0.0/0 to Network Access (allow all IPs for Vercel)
  5. Copy the connection string to MONGODB_URI
